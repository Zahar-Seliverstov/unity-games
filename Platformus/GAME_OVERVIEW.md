# Platformer — Обзор реализации

## Суть игры

2D платформер. Игрок — оранжевый квадрат, прыгает по зелёным платформам, добирается до жёлтого флага и переходит на следующий уровень. Всего 5 уровней. Два таймера: **2 минуты на уровень** и **10 минут на всю игру**. Истёк любой — начинай с уровня 1.

---

## Управление

| Клавиша | Действие |
|---------|----------|
| `A` | Движение влево |
| `D` | Движение вправо |
| `Space` | Прыжок (только стоя на поверхности) |

---

## Структура проекта

```
Assets/
  Scripts/
    GameManager.cs       — игровое состояние, таймеры, HUD, переходы между сценами
    PlayerController.cs  — движение, прыжок, физика, респавн при падении
    FlagTrigger.cs       — триггер на флаге, вызывает переход на следующий уровень
    CameraFollow.cs      — камера плавно следует за игроком
  Editor/
    LevelCreator.cs      — редакторский скрипт, строит все 5 сцен из данных
  Scenes/
    Level1.unity — Level5.unity
```

## Скрипты

### GameManager.cs

**Роль:** главный управляющий класс. Живёт с первого уровня до конца игры.

**Синглтон + DontDestroyOnLoad** — создаётся в Level1, при загрузке новой сцены не уничтожается. Если в сцене создаётся второй экземпляр (чего не происходит, но на случай ошибки) — он сразу уничтожается.

**Таймеры (Update):**
- `totalTimeRemaining` — убывает каждый кадр, стартует с 600 (10 мин)
- `levelTimeRemaining` — убывает каждый кадр, стартует с 120 (2 мин)
- Если любой ≤ 0 → `GoToLevel1()`

**Переходы:**
- `NextLevel()` — вызывается флагом. Увеличивает `currentLevel`, сбрасывает `levelTimeRemaining` до 120, загружает `"Level" + currentLevel`. Если уровень был 5 — тоже уходит на Level1.
- `GoToLevel1()` — сбрасывает всё в начальные значения и загружает Level1.

**HUD (OnGUI):** рисуется поверх игры через IMGUI (не Canvas). Полупрозрачный чёрный прямоугольник в левом верхнем углу с тремя строками: номер уровня, таймер уровня, общий таймер.

### PlayerController.cs

**Роль:** управление игроком. Висит на объекте Player.

**Требует:** `Rigidbody2D` + `BoxCollider2D` (атрибут `[RequireComponent]` гарантирует их наличие).

**Update (ввод):**
- Читает `Keyboard.current` из New Input System (пакет `com.unity.inputsystem`)
- `A/D` → записывает в `horizontalInput` (-1, 0 или +1)
- `Space` + стоит на земле → ставит флаг `jumpPressed = true`
- Флаг нужен потому что прыжок применяется в FixedUpdate, и если сделать всё в одном месте — можно пропустить нажатие между кадрами

**FixedUpdate (физика):**
1. `CheckGrounded()` — обновляет `isGrounded`
2. Устанавливает `rb.linearVelocity.x = horizontalInput * moveSpeed`, Y не трогает (гравитация работает сама)
3. Если `jumpPressed` — задаёт `linearVelocity.y = jumpForce`, сбрасывает флаг
4. Если `position.y < -12` — телепортирует на `startPosition` (спавн), обнуляет скорость

**CheckGrounded():**
Берёт `BoxCollider2D.bounds`, опускается на 0.02 ниже подошвы, создаёт тонкий горизонтальный прямоугольник (85% ширины коллайдера, высота 0.08), вызывает `Physics2D.OverlapBox`. Если в результатах есть коллайдер чужого объекта — игрок стоит на земле. Триггеры игнорируются (`useTriggers = false`), слои не фильтруются.

**Физические настройки объекта:**
- `gravityScale = 3` — падение в 3× быстрее стандартного, прыжок ощущается резче
- `freezeRotation = true` — квадрат не вращается при столкновениях
- `CollisionDetectionMode2D.Continuous` — не проваливается сквозь платформы на высокой скорости

### FlagTrigger.cs

**Роль:** компонент на объекте Flag.

Флаг — это Quad с `BoxCollider2D` у которого `isTrigger = true`. Когда в него входит коллайдер с тегом `"Player"` → вызывается `GameManager.Instance.NextLevel()`. Проверка на `null` на случай если GameManager почему-то не существует.

---

### CameraFollow.cs

**Роль:** висит на камере, плавно следует за игроком.

**LateUpdate** (выполняется после всех Update и FixedUpdate — позиция игрока уже финальная для этого кадра):
- Если `target == null` — ищет объект с тегом `"Player"` и присваивает. Это fallback на случай, если сцена загрузилась без сохранённой ссылки.
- Вычисляет желаемую позицию: `target.x + offsetX`, `target.y + offsetY`, Z камеры не меняется (-10)
- `Vector3.Lerp(current, desired, smoothSpeed * Time.deltaTime)` — плавное приближение. При `smoothSpeed = 8` камера догоняет игрока примерно за 0.12 секунды.

`offsetX = 2` — камера чуть впереди по горизонтали, `offsetY = 1.5` — чуть выше игрока.

### LevelCreator.cs (Editor)

**Роль:** редакторский инструмент. Меню `Game → Setup Platformer` создаёт все 5 сцен с нуля и регистрирует их в Build Settings.

**Данные уровней** хранятся как статические массивы:
- `L1`–`L5` — платформы, каждая: `[centerX, centerY, scaleX, scaleY]`. Первый элемент всегда земля (широкий).
- `L1Flag`–`L5Flag` — позиция флага `[x, y]`
- `L1Start`–`L5Start` — стартовая позиция игрока `[x, y]`

**Setup():** перебирает уровни 1–5, вызывает `BuildLevel()` для каждого, регистрирует сцены в `EditorBuildSettings.scenes`, открывает Level1.

**BuildLevel():** для каждой сцены:
1. Создаёт пустую сцену (`NewSceneSetup.EmptyScene`)
2. Добавляет камеру (ортографическая, размер 7, синий фон, z=-10) + `CameraFollow`
3. Добавляет направленный свет
4. Добавляет `GameManager` только в Level1 (он живёт дальше сам через DontDestroyOnLoad)
5. Создаёт игрока (Quad, тег "Player", Rigidbody2D, BoxCollider2D, PlayerController), красит оранжевым, передаёт ссылку в CameraFollow
6. Создаёт платформы из массива данных: первая — Ground (коричневый), остальные — Platform_N (зелёные)
7. Создаёт флаг (Quad, BoxCollider2D с isTrigger, FlagTrigger), красит жёлтым
8. Сохраняет сцену

**ApplyColor():** Quad-примитивы в URP 2D по умолчанию используют шейдер `Mesh2D-Lit-Default` у которого нет свойства цвета. Поэтому функция принудительно создаёт новый материал с шейдером `Universal Render Pipeline/Unlit` (у него есть `_BaseColor` и `_Color`) и присваивает его renderer'у.

## Сложность уровней

| Уровень | Платформы | Ширина платформ | Особенность |
|---------|-----------|-----------------|-------------|
| 1 | 4 | 4 юнита | Прямая лестница вверх |
| 2 | 5 | 2.5 юнита | Земля только слева, большие промежутки |
| 3 | 5 | 2.5 юнита | Сильный набор высоты |
| 4 | 7 | 2 юнита | Зигзаг: сначала вниз, потом обратно вверх |
| 5 | 8 | 1.5 юнита | Узкие платформы, обратные ходы |

## Как пересоздать уровни

Меню Unity: **Game → Setup Platformer**

Это пересоздаёт все 5 сцен с нуля (перезаписывает существующие).

## Технические детали

- **Движок:** Unity 6 (6000.4.3f1)
- **Рендер:** URP 17.4.0 (2D профиль)
- **Ввод:** New Input System 1.19.0 (`Keyboard.current`)
- **Физика:** 2D (Rigidbody2D, BoxCollider2D, Physics2D.OverlapBox)
- **HUD:** IMGUI (OnGUI в GameManager), не Canvas — без зависимостей от UI-пакетов
- **Переходы:** SceneManager.LoadScene по имени (`"Level1"` — `"Level5"`)
